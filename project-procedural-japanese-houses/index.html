Houdini Dev Log: Procedural Japanese Houses

About

After taking the Procedural Lake Houses course by Anastasia Opara, I wanted to challenge myself by creating something similar with a different theme. I applied the methodologies learned in the course to generate traditional Japanese houses procedurally.

At the same time, I used this project as an opportunity to explore different workflows for procedural modeling across various modeling and texturing software, before bringing the final result into Unreal Engine.

Key Objectives

  • Deepening my understanding of VEX.
  • Expanding my knowledge of procedural generation in Houdini.
  • Developing complex procedural models.
  • Establishing efficient workflows between different modeling and texturing software.
  • Integrating the final result into Unreal Engine.

Searching for the Silhouette

In my opinion, one of the most challenging aspects of developing procedural modeling is the beginning. At this stage, when nothing is defined yet or how it is going to be achieved, a wide range of possibilities opens up. Without a clear path, it‚Äôs easy to become overwhelmed. So‚Ä¶ where do we start?

Procedural modeling is still modeling after all, so we can start in the same way as we would when modeling a conventional 3D object. First, we establish key objectives that will guide the development, and then we gather references to help define the idea we have in mind. For this project, there were two elements I wanted: the buildings should feature traditional Chinese-Japanese architecture, and they should fit into a snowy environment.

When creating a mood board, I keep a few things in mind. First, I look for a couple of images that resemble the result I‚Äôm aiming for. Then, I identify the key elements that make up these images and create separate islands with references for each of them. As the project progresses, I continue adding references for details and materials to refine the final look.

At this point, I have an answer to the question: what to do, but the hardest question still remains: how to do it?

This creative process often relies on ‚Äúhappy ideas‚Äù. In my case, I started by researching how these types of buildings are built. While studying blueprints of traditional houses, I noticed that they all commonly use tatami mats. The arrangement of these tatami mats creates square-shaped rooms, which can be easily generated by instancing cubes on a grid in Houdini.

Without a doubt, the most distinctive element of traditional Japanese buildings is their roofs. In addition to being the most complex component of the structure, they are also what gives the model its strongest identity. That‚Äôs why I took the time to carefully study these types of roofs. But for now, the only relevant aspects when defining the silhouette or blocking are the following points:

  • A traditional Japanese building always has a main roof at the highest point, with secondary roofs positioned below it.

  • Roof surfaces should be grouped together, always aiming to form the largest continuous roof area possible.

  • Simple roofs can be connected to create corners instead of being placed at different heights.

  • Main Types of Roofs:

    • Kirizuma ‚Äì Gabled roofs.

    • Hogyo ‚Äì Pyramid-shaped roofs with a square base, often adorned with an ornamental element at the top.

    • Yosemune (or Shich≈´) ‚Äì Hipped roofs.

    • Irimoya ‚Äì A combination of gabled and hipped roofs.

The boxes and planes that form the roofs are still far from the final result, but they provide a solid foundation to start working from. At this point, I expose several variables to play with the procedural model, check its consistency, and bugfix errors.

Terrace Workflow:

Houdini ‚Üí Unreal Engine 5

Terraces are an element that appears repeatedly in my references. In particular, I was drawn to the patterns formed by the railings. If we had to model this with any conventional modeling software, creating 10 variations of railings would easily take one or two days. However, by using Houdini, we can generate many more variations in just a few minutes. The secret? The Trace node.

Through 2D images, we can generate a plane that creates geometry with the pattern‚Äôs shape. It is then extruded to add volume, and a couple of primitives are added to form a frame. As a result, we get traditional Japanese procedural railings, ready to be added to the house.

Building upon the foundation created earlier, we establish a set of rules to determine the points where the railings will be instantiated, along with their rotation and scale. We use the Copy to Points node to place the railings on the house structure. Finally, we add additional information to the model that will be used in Unreal Engine, such as the material it will use, or a value stored in the red channel of the vertex color, which will be used as a mask to create color variations in the wood.

Doors and Windows Workflow:

Belnder ‚Üí Houdini ‚Üí Unreal Engine 5

One of the biggest advantages of procedural modeling is the flexibility it offers when iterating on the modeling workflow. Typically, when creating a conventional 3D model, we follow the same steps: blocking, high-poly modeling, retopology, UV mapping, baking textures, texturing, and integrating into the engine. Procedural modeling follows a similar process, but it allows for shortcuts that speed up development and provide the ability to go back at any time to make adjustments.

To add doors and windows to the house structure, the first step is to create a base model in any 3D modeling software. We assign materials such as washi paper and wood for the frames. These models use a trim sheet material, so there is no need to create a high-resolution version or bake textures.

Once the mesh is imported into Houdini, I apply a custom node I created specifically for this project, called ‚Äúset_trim_uvs‚Äù. This node automatically generates UVs and correctly places them within the trim sheet space. Finally, additional data is added to be used in Unreal Engine materials, making the model ready for integration.

As mentioned in the previous section, procedural rules determine the exact points where these assets will be placed. Before adding doors and windows, these points are also used to create holes in the house‚Äôs structure using Boolean operations.

The main advantage of this workflow is that doors and windows can be modified at any point during development. This allows for quick and effortless adjustments, making the modeling process much more efficient.

Roofs and Columns Workflow:

Belnder ‚Üí ZBrush ‚Üí Substance Painter ‚Üí Houdini ‚Üí Unreal Engine 5

To complete the piece, I added a few elements modeled in ZBrush. In this case, the workflow follows a more conventional approach. It is quite similar to creating a modular set, as seen with the roofs or columns. However, incorporating Houdini into the workflow allows us to define procedural rules and use nodes to better integrate these modular pieces.

For example, to create the columns, I only modeled two rows of bricks. Houdini then handles the rotation and scaling, making them feel like a complete, cohesive structure. As with the previous workflows, additional data is added to be used in Unreal Engine materials, helping to break symmetry and create a more unique and natural-looking element.
